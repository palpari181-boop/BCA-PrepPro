📘 Chapter 1 : Introduction to Data Structures

Data Structure is one of the most important concepts in Computer Science and Programming. It provides a systematic way to organize, store, manage, and process data efficiently. Whenever a programmer develops software, websites, mobile applications, operating systems, games, or databases, data structures play a vital role in improving performance and reducing execution time.

In simple words, a Data Structure is a method of organizing data inside the computer's memory so that operations like searching, inserting, deleting, updating, and sorting can be performed quickly and efficiently.

📖 Definition

A Data Structure is a specialized format used to organize and store data in a computer so that it can be accessed, modified, and processed efficiently whenever required.

💡 Understanding Data Structure with a Real-Life Example

Imagine a library containing thousands of books. If all books are placed randomly on the floor, finding a single book may take several hours. However, if books are arranged shelf-wise according to subjects, authors, or serial numbers, the required book can be found within minutes.

Similarly, computers store millions of pieces of information. Without proper organization, processing data becomes slow and inefficient. Data Structures help organize this information in an efficient manner.

Example:
  • 📚 Library → Books are arranged shelf-wise.
  • 🏥 Hospital → Patient records are arranged department-wise.
  • 🏦 Bank → Customer information is stored systematically.
  • 📱 Mobile Phone → Contact list is stored alphabetically.
  • 🛒 Shopping Website → Products are categorized for faster searching.

📚 Types of Data Structures

Data Structures can be classified into different categories depending on the way data is stored, organized, and accessed. Understanding these classifications is very important because every programming problem requires a suitable data structure.

The major classification of Data Structures is shown below:

  • ✅ Primitive Data Structures
  • ✅ Non-Primitive Data Structures

🔹 Primitive Data Structures

Primitive Data Structures are the basic and built-in data types provided by programming languages. These are directly supported by the compiler and are used to store single values.

They act as the building blocks for all advanced data structures. Without primitive data types, complex data structures cannot be created.

Characteristics of Primitive Data Structures

  • ✔ Built into the programming language.
  • ✔ Stores only one value at a time.
  • ✔ Requires fixed memory.
  • ✔ Easy to declare and use.
  • ✔ Faster execution.
  • ✔ Used in almost every program.

Examples of Primitive Data Types

Data Type Description Example
Integer (int) Stores whole numbers. 15, 200, -10
Float Stores decimal values. 3.14, 25.50
Double Stores high precision decimal values. 123.456789
Character (char) Stores a single character. 'A', 'Z'
Boolean Stores True or False. true / false

Example

int age = 20;
float salary = 25000.50;
char grade = 'A';
bool pass = true;

🔷 Non-Primitive Data Structures

Non-Primitive Data Structures are advanced data structures that are created using primitive data types. They can store multiple values and represent complex relationships among data.

These data structures are generally designed by programmers to solve real-world problems.

Main Characteristics

  • ✔ Can store multiple values.
  • ✔ Can represent complex data.
  • ✔ Memory can be static or dynamic.
  • ✔ Used in software development.
  • ✔ Makes searching and processing easier.

Non-Primitive Data Structures are divided into two categories:

  • 📌 Linear Data Structures
  • 📌 Non-Linear Data Structures

📖 Linear Data Structures

A Linear Data Structure stores data elements in a sequential order. Each element is connected to the next element in a single line.

Traversal is straightforward because data is arranged one after another.

Characteristics

  • ✔ Sequential arrangement.
  • ✔ Easy traversal.
  • ✔ One-to-one relationship.
  • ✔ Simple implementation.

Examples

These structures are mainly used when relationships between data are more complex.

📦 Static Data Structures

A Static Data Structure is a type of data structure in which the memory size is fixed before the execution of the program. Once the memory is allocated, its size cannot be increased or decreased during runtime.

Static data structures are simple to implement and execute faster because the memory is allocated only once. However, if more memory is required later, it cannot be allocated, which may lead to memory wastage or overflow.

Characteristics of Static Data Structures

  • ✔ Fixed memory allocation.
  • ✔ Size is decided before program execution.
  • ✔ Faster data access.
  • ✔ Easy implementation.
  • ✔ Suitable for small applications.
  • ✔ Less flexible.

Advantages

  • ✔ Simple to understand.
  • ✔ Faster execution.
  • ✔ Easy memory management.
  • ✔ Direct access to elements.

Disadvantages

  • ❌ Memory cannot be changed.
  • ❌ Memory wastage is possible.
  • ❌ Not suitable for large applications.
  • ❌ Cannot store unlimited data.
Example:

An array of size 100 is created.
Even if only 10 elements are used, memory for all 100 elements remains reserved.

⚡ Dynamic Data Structures

Dynamic Data Structures allocate memory during program execution. The memory size can increase or decrease whenever required. This makes dynamic structures flexible and memory efficient.

Dynamic memory allocation is performed using pointers. Examples include Linked List, Trees, and Graphs.

Characteristics

  • ✔ Memory allocated at runtime.
  • ✔ Flexible size.
  • ✔ Better memory utilization.
  • ✔ Suitable for large applications.
  • ✔ Can grow or shrink anytime.

Advantages

  • ✔ No memory wastage.
  • ✔ Flexible.
  • ✔ Better performance for changing data.
  • ✔ Can store unlimited records (depending on available memory).

Disadvantages

  • ❌ More complex than static structures.
  • ❌ Uses pointers.
  • ❌ Slightly slower than static structures.
  • ❌ Programming becomes difficult.
Examples of Dynamic Data Structures
  • 📄 Linked List

⚙️ Operations on Data Structures

Every data structure supports different operations that allow programmers to manipulate the stored data efficiently.

The six basic operations are:

  1. Traversal
  2. Insertion
  3. Deletion
  4. Searching
  5. Sorting
  6. Merging

1️⃣ Traversal

Traversal means visiting each element of the data structure exactly once. It is mainly used for displaying, processing, or checking all stored elements.

Example: Printing all student names stored in an array.

2️⃣ Insertion

Insertion means adding a new element into the data structure. The new element may be inserted at the beginning, middle, or end depending on the requirement.

Example: Adding a new employee record into the database.

3️⃣ Deletion

Deletion is the process of removing an existing element from the data structure. Proper deletion helps in maintaining correct and updated information.

Example: Removing a cancelled train ticket record.

4️⃣ Searching

Searching is used to locate a specific element within the data structure. Efficient searching algorithms reduce execution time significantly.

Example: Searching a contact number in a mobile phone.

5️⃣ Sorting

Sorting means arranging data in ascending or descending order. Sorting improves searching speed and overall efficiency.

Example: Arranging student marks from highest to lowest.

6️⃣ Merging

Merging combines two or more data structures into a single structure. This operation is useful while combining datasets.

Example: Combining two class attendance lists.

💡 Important Point

Almost every Data Structure supports these six operations. However, the implementation method differs from one structure to another.

⏱️ Time Complexity

Time Complexity is the amount of time required by an algorithm to complete its execution. It tells us how the running time of a program increases when the size of input data increases. Time Complexity does not measure the actual time in seconds. Instead, it measures the number of basic operations performed by an algorithm.

For example, if an algorithm takes 10 operations for 10 elements and 100 operations for 100 elements, we say that its running time grows with the size of input.

Why Time Complexity is Important?

  • ✔ Helps compare different algorithms.
  • ✔ Improves program performance.
  • ✔ Reduces execution time.
  • ✔ Helps choose the best algorithm for large datasets.
  • ✔ Essential in competitive programming and software development.

📈 Big O Notation (O)

Big O Notation represents the worst-case performance of an algorithm. It tells us the maximum amount of time an algorithm may take to complete its execution.

Example: Searching an element in an unsorted array may require checking every element. Therefore, the worst-case complexity is O(n).

Notation Name Performance
O(1) Constant Time Excellent
O(log n) Logarithmic Very Fast
O(n) Linear Good
O(n log n) Linear Logarithmic Efficient
O(n²) Quadratic Slow for large data

📉 Omega Notation (Ω)

Omega Notation represents the best-case performance of an algorithm. It indicates the minimum amount of time required by an algorithm.

Example: Searching the first element of an array requires only one comparison. Hence, the best case is Ω(1).

📊 Theta Notation (Θ)

Theta Notation represents the average-case performance. It gives the average running time considering all possible inputs.

Easy Trick to Remember

Big O → Worst Case 🚨
Omega → Best Case ⚡
Theta → Average Case 📊

💾 Space Complexity

Space Complexity refers to the amount of memory required by an algorithm during execution. It includes both input memory and extra memory used by variables, arrays, recursion, etc.

A good algorithm should use minimum memory while maintaining good performance.

Importance of Space Complexity

  • ✔ Reduces memory consumption.
  • ✔ Makes software efficient.
  • ✔ Improves system performance.
  • ✔ Prevents memory overflow.
  • ✔ Useful for mobile and embedded systems.

📘 Chapter 2 : Arrays

Array is one of the most fundamental and widely used data structures in computer science. It is used to store multiple values of the same data type in a single variable. Instead of creating many individual variables, an array allows us to store all related elements together, making programs more organized, readable, and efficient.

Arrays are used in almost every programming language such as C, C++, Java, Python, and C#. Many advanced data structures like Stacks, Queues, Heaps, and Hash Tables are also implemented using arrays.

📖 Definition of Array

An Array is a collection of elements of the same data type stored in contiguous (consecutive) memory locations. Each element in an array is identified by its index number, which starts from 0.

💡 Why Do We Need Arrays?

Imagine a teacher wants to store the marks of 100 students. Without arrays, the programmer would have to create 100 different variables such as:

marks1
marks2
marks3
marks4
...
marks100

Managing such a large number of variables becomes difficult and increases the chances of errors. Arrays solve this problem by allowing all values to be stored under one variable name.

Example:

int marks[100];

Now all 100 student marks can be stored inside a single array, and each value can be accessed using its index number.

🌍 Real-Life Examples of Arrays

Arrays are used in many real-life situations where similar types of data need to be stored together.

  • 🎓 Marks of students in a class.
  • 💰 Monthly salary of employees.
  • 🌡️ Daily temperature of a city.
  • 📱 Contact numbers stored in a mobile application.
  • 🏏 Runs scored by players in a cricket match.
  • 🛒 Prices of products in an online shopping website.
  • 📚 Roll numbers of students.
  • 🎵 Playlist of songs.

⭐ Characteristics of Arrays

Arrays have several important characteristics that make them useful for storing data.

  • ✔ Stores elements of the same data type.
  • ✔ Elements are stored in continuous memory locations.
  • ✔ Every element has a unique index.
  • ✔ Indexing starts from 0.
  • ✔ Random access is possible.
  • ✔ Fixed size after declaration.
  • ✔ Easy traversal using loops.
  • ✔ Fast access to elements.

💡 Important Note

The size of an array cannot be changed after it has been declared. If more space is required, a new array must be created.

🧠 How Array Works?

When an array is declared, the computer reserves a block of continuous memory. Each element occupies equal memory depending on its data type.

Example:

int numbers[5] = {10,20,30,40,50};

Memory Representation:

Index Value
0 10
1 20
2 30
3 40
4 50

To access the third element, we write:

numbers[2]

Since indexing starts from 0, numbers[2] returns the value 30.

📝 Declaration of Arrays

Before storing values inside an array, it must be declared. Array declaration tells the compiler about the data type, array name, and the maximum number of elements that can be stored. Memory for all elements is reserved during declaration.

General Syntax


data_type array_name[size];

Examples


int marks[50];

float salary[20];

char name[30];

double price[15];

In the above examples:

  • int → Data type of the array.
  • marks → Name of the array.
  • 50 → Maximum number of elements.

🎯 Rules for Declaring an Array

Some important rules should always be followed while declaring arrays.

  • ✔ Array name must be a valid identifier.
  • ✔ Array size should be a positive integer.
  • ✔ All elements must belong to the same data type.
  • ✔ Index always starts from 0.
  • ✔ Size cannot be negative.
  • ✔ Memory size becomes fixed after declaration.
  • ✔ Array size should not exceed available memory.

Exam Tip

📚 Initialization of Arrays

Initialization means assigning values to array elements. Initialization can be performed during declaration or after declaration.

Method 1 : Initialization During Declaration


int marks[5] = {90,85,78,88,95};

The compiler stores all five values automatically in continuous memory locations.

Method 2 : Partial Initialization


int numbers[5]={10,20};

Only the first two elements receive values. The remaining elements automatically become 0.

Index Stored Value
0 10
1 20
2 0
3 0
4 0

Method 3 : Size Automatically Determined


int numbers[]={5,10,15,20,25};

Here, the compiler automatically calculates the array size based on the number of elements. In this example, the size becomes 5.

Method 4 : Initialization After Declaration


int marks[5];

marks[0]=85;
marks[1]=92;
marks[2]=78;
marks[3]=88;
marks[4]=95;

Each element is assigned individually using its index number.

💾 Memory Representation of Arrays

One of the biggest advantages of arrays is that all elements are stored in contiguous (consecutive) memory locations. This allows the processor to access any element directly without searching the entire collection.

Example


int A[5]={10,20,30,40,50};
Index Value Memory Address (Example)

Each integer occupies 4 bytes of memory (on many systems). Therefore, addresses increase by 4 bytes for every next element.

Important Formula

Address of Element = Base Address + (Index × Size of Data Type)

🔍 Accessing Array Elements

Accessing an array element means retrieving or reading the value stored at a particular position in the array. Every element of an array is identified by an index number. Since array indexing starts from 0, the first element is stored at index 0, the second element at index 1, and so on.

📌 Syntax


array_name[index];

Example


int marks[5] = {80, 85, 90, 75, 95};

printf("%d", marks[0]);   // Output: 80

printf("%d", marks[3]);   // Output: 75

Using the correct index allows the program to directly access any element without checking other elements. This direct access makes arrays very fast compared to many other data structures.

✍️ Reading Values into an Array

Values can also be entered by the user during program execution. This is known as runtime input. The most common method is using a for loop.

for(int i=0; i<5; i++)
{
    scanf("%d", &marks[i]);
}

Each value entered by the user is stored at its respective index.

1️⃣ Traversing an Array

Traversal means visiting each element of an array exactly once. It is the simplest and most commonly used operation. Traversal is mainly used to display data, calculate sum, average, largest element, smallest element, and perform many other calculations.

Algorithm

  1. Start from index 0.
  2. Visit every element one by one.
  3. Repeat until the last element.
  4. Stop.

C Program

{
int arr[5]={10,20,30,40,50};

for(int i=0;i<5;i++)
{
printf("%d ",arr[i]);
}

return 0;
}

Output: 10 20 30 40 50

Applications

  • Displaying student marks
  • Calculating total salary
  • Finding average marks
  • Printing product prices
  • Reading sensor values

🔍 Searching and Sorting Techniques

Searching and Sorting are two of the most important operations performed on arrays and other data structures. Searching helps us find a required element, whereas Sorting arranges data in ascending or descending order. These techniques improve efficiency and make data management easier.

🔍 Searching

Searching is the process of finding the location of a particular element in a collection of data. If the element is found, its position is returned; otherwise, the search reports that the element does not exist.

Types of Searching

  • ✔ Linear Search
  • ✔ Binary Search

1️⃣ Linear Search

Linear Search is the simplest searching technique. It checks every element one by one until the required element is found or the array ends.

Algorithm

  1. Start from the first element.
  2. Compare each element with the required value.
  3. If matched, return the position.
  4. If the last element is reached, stop the search.

C Program


int key = 30;

for(int i=0;i<5;i++)
{
    if(arr[i]==key)
    {
        printf("Element Found");
        break;
    }
}

Time Complexity: O(n)

2️⃣ Binary Search

Binary Search is a faster searching technique that works only on sorted arrays. It repeatedly divides the array into two equal parts until the element is found.

Conditions

  • ✔ Array must be sorted.
  • ✔ Middle element is compared first.
  • ✔ Search continues in only one half.

Time Complexity: O(log n)

📊 Difference Between Linear and Binary Search

Linear Search Binary Search
Works on sorted and unsorted arrays. Works only on sorted arrays.
Simple algorithm. More efficient.
O(n) O(log n)

🔄 Sorting

Sorting is the process of arranging elements in ascending or descending order. It makes searching faster and improves data organization.

Common Sorting Techniques

  • ✔ Bubble Sort
  • ✔ Selection Sort
  • ✔ Insertion Sort

1️⃣ Bubble Sort

Bubble Sort repeatedly compares adjacent elements and swaps them if they are in the wrong order. After each pass, the largest element moves to its correct position. 


for(i=0;iarr[j+1])
      {
         temp=arr[j];
         arr[j]=arr[j+1];
         arr[j+1]=temp;
      }
   }
}

Time Complexity: O(n²)

2️⃣ Selection Sort

Selection Sort selects the smallest element from the unsorted part of the array and places it at the correct position.

Time Complexity: O(n²)

3️⃣ Insertion Sort

Insertion Sort inserts each element into its correct position in the already sorted part of the array. It is efficient for small datasets.

Time Complexity: O(n²)

🔗 Chapter 3 : Linked List

A Linked List is a dynamic linear data structure in which elements are called nodes. Unlike arrays, linked list elements are not stored in contiguous memory locations. Each node contains two parts: the first part stores the data, while the second part stores the address (pointer) of the next node. Because of this structure, linked lists can grow or shrink dynamically during program execution.

📖 Definition

A Linked List is a collection of nodes connected through pointers, where each node contains data and the address of the next node.

🤔 Why Linked List is Needed?

Arrays have a fixed size and require contiguous memory. If we need to insert or delete elements frequently, arrays become inefficient because elements have to be shifted. Linked Lists solve this problem by allowing dynamic memory allocation and easy insertion or deletion without shifting elements.

  • ✔ Dynamic memory allocation.
  • ✔ Easy insertion and deletion.
  • ✔ Efficient memory utilization.
  • ✔ No memory wastage due to fixed size.

🧩 Structure of a Node

Each node in a linked list consists of two parts:

Part Description
Data Stores the actual value.
Next Pointer Stores the address of the next node.

| Data | Next Address |
-------------------------

⚙️ Working of Linked List

The first node is called the Head Node. The last node stores NULL in its next pointer, indicating the end of the list.

1️⃣ Singly Linked List

A Singly Linked List is the simplest type of linked list. Each node contains data and one pointer that points to the next node. Traversal is possible only in the forward direction.

Advantages

  • ✔ Less memory required.
  • ✔ Dynamic size.

2️⃣ Doubly Linked List

In a Doubly Linked List, every node contains two pointers: one points to the previous node and the other points to the next node. This allows traversal in both forward and backward directions. 


NULL <- 10 <-> 20 <-> 30 -> NULL

In a Circular Linked List, the last node points back to the first node instead of NULL. This forms a complete circle and allows continuous traversal.

4️⃣ Circular Doubly Linked List

It combines the features of both Doubly Linked List and Circular Linked List. Every node has two pointers, and the last node connects back to the first node.

3. Priority Queue

Elements are removed according to their priority instead of arrival time. Higher priority elements are processed first.

4. Double Ended Queue (Deque)

Insertion and deletion can be performed from both Front and Rear ends.

🌐 Chapter 7 : Graph

A Graph is a non-linear data structure that consists of a collection of vertices (nodes) and edges. Unlike trees, a graph does not have a root node and can represent complex relationships between different objects. Graphs are one of the most important data structures because they are used in social media, GPS navigation, computer networks, recommendation systems, artificial intelligence, and many other real-world applications.

📖 Definition

A Graph is a collection of vertices (V) and edges (E), where vertices represent objects and edges represent the relationships or connections between those objects.

📚 Terminologies

Cycle A path that starts and ends at the same vertex.
Connected Graph Every vertex is reachable from every other vertex.

🔍 Breadth First Search (BFS)

Breadth First Search (BFS) visits nodes level by level. It uses a Queue data structure to keep track of the next vertex to visit. BFS always explores all neighboring vertices before moving to the next level.

Algorithm

  1. Start from the source vertex.
  2. Mark it as visited.
  3. Insert it into the Queue.
  4. Remove a vertex from the Queue.
  5. Visit all unvisited neighbors.
  6. Repeat until the Queue becomes empty.

Time Complexity: O(V + E)

🌲 Depth First Search (DFS)

Depth First Search (DFS) explores one branch completely before moving to another branch. It uses a Stack or recursion.

Algorithm

  1. Start from the source vertex.
  2. Mark it as visited.
  3. Visit an unvisited adjacent vertex.
  4. Continue until no unvisited vertex remains.
  5. Backtrack and continue.

Time Complexity: O(V + E)

📊 Difference Between BFS and DFS
