🗄️ Chapter 1 : Introduction to Database Management System (DBMS)

A Database Management System (DBMS) is software that allows users to create, store, organize, retrieve, update, and manage data efficiently. Instead of storing information in separate files, a DBMS stores data in a structured database, making it easy to access, modify, and secure. Today almost every organization such as banks, hospitals, schools, railway reservation systems, shopping websites, and social media platforms uses DBMS.

📖 Definition

A Database Management System (DBMS) is software that acts as an interface between the user and the database. It enables users to perform operations such as inserting, updating, deleting, and retrieving data efficiently.

📚 What is Data?

Data refers to raw facts and figures that have not yet been processed. For example:

  • Student Name
  • Roll Number
  • Age
  • Marks
  • Salary

Raw data becomes useful information after processing.

📂 What is a Database?

A Database is an organized collection of related data. It stores information in a structured manner so that it can be easily accessed, managed, and updated whenever required.

🏛️ DBMS Architecture

DBMS Architecture defines how users interact with the database and how data is processed. The architecture improves security, data independence, and overall performance. Based on the number of layers, DBMS architecture is mainly classified into three types:

  • ✔ One-Tier Architecture
  • ✔ Two-Tier Architecture
  • ✔ Three-Tier Architecture

1️⃣ One-Tier Architecture

In One-Tier Architecture, the user, application, and database are all present on the same system. There is no network communication. It is mainly used for learning and testing purposes. 



+--------------------------+
|        USER              |
|  Application + Database  |
+--------------------------+

Advantages

  • ✔ Simple to use.
  • ✔ Fast execution.
  • ✔ No network required.

Disadvantages

  • ❌ Poor security.
  • ❌ Not suitable for large organizations.

2️⃣ Two-Tier Architecture

In Two-Tier Architecture, the client directly communicates with the database server. The client sends requests and receives responses from the database. 



Client  ⇄  Database Server

Advantages

  • ✔ Faster communication.
  • ✔ Easy implementation.
  • ✔ Better than One-Tier.

Disadvantages

  • ❌ Limited scalability.
  • ❌ Less secure than Three-Tier.

3️⃣ Three-Tier Architecture

Three-Tier Architecture is the most widely used architecture in modern applications. It consists of Presentation Layer, Application Layer, and Database Layer. The client never communicates directly with the database. 



+----------------------+
|     Client/User      |
+----------+-----------+
           |
           |
+----------v-----------+
|  Application Server  |
+----------+-----------+
           |
           |
+----------v-----------+
|   Database Server    |
+----------------------+

Advantages

  • ✔ High Security
  • ✔ Easy Maintenance
  • ✔ Better Performance
  • ✔ Scalable
  • ✔ Used in Real Applications

🔷 Chapter 2 : Entity Relationship (ER) Model

The Entity Relationship (ER) Model is one of the most important concepts in Database Management Systems. It is a high-level conceptual model used to design the structure of a database before implementation. The ER Model represents real-world objects, their properties, and the relationships between them in the form of an ER Diagram. It was introduced by Peter Chen in 1976 and is widely used in database designing because it provides a clear graphical representation of data.

📖 Definition

The Entity Relationship Model is a conceptual database model that describes data using Entities, Attributes, and Relationships. It helps database designers understand and organize data before creating database tables.

🎯 Objectives of ER Model

  • ✔ To represent real-world objects.
  • ✔ To simplify database design.
  • ✔ To identify relationships between entities.
  • ✔ To reduce redundancy during database creation.
  • ✔ To create a blueprint before implementation.

🏷️ Entity

An Entity is any real-world object, person, place, thing, or event that can be identified independently. Each entity contains information that needs to be stored in the database.

Examples of Entities

  • 👨 Student
  • 👩 Teacher
  • 🏢 Department
  • 📚 Book
  • 🏦 Bank Account
  • 🛒 Customer
  • 📦 Product

📝 Attributes

Attributes describe the properties or characteristics of an entity. Each entity contains one or more attributes.

Entity Attributes
Student ID, Name, Age, Course
Employee Emp_ID, Name, Salary
Book Book_ID, Title, Author

📂 Types of Attributes

1. Simple Attribute

Cannot be divided further. Example: Age, Gender.

2. Composite Attribute

Can be divided into sub-parts. Example: Address → City, State, PIN Code.

3. Single-Valued Attribute

Contains only one value. Example: Roll Number.

4. Multi-Valued Attribute

Contains multiple values. Example: Phone Numbers.

5. Derived Attribute

Calculated from another attribute. Example: Age calculated from Date of Birth.

🔑 Chapter 2.1 : Keys in DBMS & ER Diagram Symbols

In a database, millions of records may be stored in different tables. To identify each record uniquely and maintain relationships between tables, DBMS uses different types of Keys. Keys play a very important role in ensuring data accuracy, preventing duplicate records, and establishing relationships between tables. Without keys, it becomes difficult to retrieve, update, or delete specific records efficiently.

📖 Definition of Key

A Key is one or more attributes used to identify a record uniquely in a database table. Keys also help in creating relationships between different tables.

🎯 Why are Keys Important?

  • ✔ Identify each record uniquely.
  • ✔ Prevent duplicate data.
  • ✔ Create relationships between tables.
  • ✔ Improve searching speed.
  • ✔ Maintain data integrity.
  • ✔ Simplify database management.

🔷 Types of Keys

  • 🔑 Super Key
  • 🔑 Candidate Key
  • 🔑 Primary Key
  • 🔑 Alternate Key
  • 🔑 Composite Key
  • 🔑 Foreign Key

1️⃣ Super Key

A Super Key is a set of one or more attributes that can uniquely identify a record in a table. It may contain extra attributes in addition to the minimum required attributes.

Example 


Student Table

Student_ID
Name
Email

Possible Super Keys

Student_ID

Student_ID + Name

Student_ID + Email

Student_ID + Name + Email

2️⃣ Candidate Key

A Candidate Key is the minimum set of attributes required to uniquely identify a record. It does not contain unnecessary attributes. A table may have multiple Candidate Keys.

Example: Student_ID Email Both can uniquely identify students.

3️⃣ Primary Key

A Primary Key is the Candidate Key selected to uniquely identify every record. It cannot contain NULL values or duplicate values. Each table has only one Primary Key.

Student_ID Name Course
101 Rahul BCA
102 Priya BCA

✔ Unique ✔ Not NULL ✔ Only One Primary Key

4️⃣ Alternate Key

Candidate Keys that are not selected as the Primary Key are called Alternate Keys. They also uniquely identify records but are not chosen as the main identifier.

Example: Primary Key = Student_ID Alternate Key = Email

5️⃣ Composite Key

A Composite Key is formed by combining two or more attributes to uniquely identify a record. Neither attribute alone is sufficient. 



Student_ID + Subject_Code

101 + CS101

6️⃣ Foreign Key

A Foreign Key is an attribute that creates a relationship between two tables. It refers to the Primary Key of another table. Foreign Keys maintain Referential Integrity.

📊 Chapter 3 : Relational Model

The Relational Model is one of the most widely used database models in Database Management System (DBMS). It was proposed by Dr. E. F. Codd in 1970. In this model, data is stored in the form of tables called Relations. Each table contains rows and columns, making data easy to understand, organize, and manage. Most modern database systems such as MySQL, Oracle, SQL Server, and PostgreSQL are based on the Relational Model.

📖 Definition

A Relational Model is a database model that stores data in the form of tables (Relations). Each table consists of rows (Tuples) and columns (Attributes), and relationships between tables are established using Keys.

⭐ Features of Relational Model

  • ✔ Data is stored in tabular form.
  • ✔ Easy to understand and maintain.
  • ✔ Supports relationships between tables.
  • ✔ Reduces data redundancy.
  • ✔ Provides high data integrity.
  • ✔ Supports SQL for data manipulation.
  • ✔ Multiple users can access data simultaneously.

📋 Basic Terminology

1️⃣ Relation

A Relation is simply a table in a database. It consists of rows and columns. Each relation stores information about a particular entity.

Example:
Student_ID Name Course
101 Rahul BCA
102 Priya BCA
103 Aman BCA

2️⃣ Tuple (Row)

A Tuple represents a single record (row) in a relation. Each tuple stores complete information about one entity.

Example: Student_ID = 101 Name = Rahul Course = BCA This complete row is called a Tuple.

3️⃣ Attribute (Column)

An Attribute represents a column of a table. It defines a particular property of an entity. Each column stores one type of information.

Attribute Description
Student_ID Unique student number
Name Name of Student
Course Course Name
Age Student Age

4️⃣ Domain

A Domain is the set of all valid values that an attribute can contain. It ensures that only meaningful data is stored in the database.

Attribute Domain
Age 17 – 60
Gender Male, Female, Other
Marks 0 – 100

5️⃣ Degree

Degree is the total number of attributes (columns) present in a relation. If a table contains four columns, then its degree is four. 



Student_ID | Name | Course | Age

Degree = 4

6️⃣ Cardinality

Cardinality is the total number of tuples (rows) present in a relation. If a table contains 100 records, then its cardinality is 100. 



Student Table

100 Rows

Cardinality = 100

🔒 Integrity Constraints

Integrity Constraints are a set of rules that ensure the accuracy, consistency, and reliability of data stored in a relational database. These rules prevent invalid data from entering the database and help maintain data quality.

📖 Definition

Integrity Constraints are rules applied on database tables to ensure that the stored data remains valid, accurate, and consistent throughout the life of the database.

📚 Types of Integrity Constraints

1️⃣ Entity Integrity

Entity Integrity states that every table must have a Primary Key, and the Primary Key value can never be NULL. This ensures that every record in the table is uniquely identifiable.

📌 Data Independence

Data Independence is the ability to change the database structure without affecting application programs. It makes database maintenance easy and flexible.

Types

  • Physical Data Independence – Changes in physical storage do not affect the logical structure.
  • Logical Data Independence – Changes in logical structure do not affect user applications.

💻 Chapter 4 : Structured Query Language (SQL)

Structured Query Language (SQL) is the standard language used to communicate with a Relational Database Management System (RDBMS). SQL allows users to create databases, store records, retrieve information, update existing data, and delete unwanted records. Almost every modern database such as MySQL, Oracle, SQL Server and PostgreSQL supports SQL.

📖 Definition

SQL (Structured Query Language) is a standard programming language used to create, manage, manipulate and retrieve data from relational databases.

⭐ Features of SQL

  • ✔ Easy to learn and use.
  • ✔ Standard language for databases.
  • ✔ Supports data retrieval and manipulation.
  • ✔ Works with multiple database systems.
  • ✔ Provides security using user privileges.
  • ✔ Supports transactions and data integrity.
  • ✔ Fast and efficient for handling large data.

📂 Types of SQL Commands

Command Type Purpose
DDL Defines database structure.
DML Manipulates records.
DQL Retrieves data.
DCL Controls user permissions.
TCL Manages transactions.

🏗️ DDL (Data Definition Language)

DDL commands are used to create and modify the structure of database objects like tables, views and indexes.

1. CREATE

Creates a new table or database.



CREATE TABLE Student(

Student_ID INT PRIMARY KEY,

Name VARCHAR(50),

Course VARCHAR(30),

Age INT

);

2. ALTER

Used to modify the structure of an existing table.



ALTER TABLE Student

ADD Email VARCHAR(50);

3. DROP

Deletes an entire table permanently.



DROP TABLE Student;

4. TRUNCATE

Deletes all rows from a table but keeps the table structure.



TRUNCATE TABLE Student;

📝 DML (Data Manipulation Language)

DML commands are used to insert, update and delete records stored inside a table.

1. INSERT



INSERT INTO Student

VALUES

(101,'Rahul','BCA',19);

2. UPDATE



UPDATE Student

SET Age=20

WHERE Student_ID=101;

3. DELETE



DELETE FROM Student

WHERE Student_ID=101;

🔍 DQL (Data Query Language)

DQL is mainly used to retrieve information from the database. The most commonly used command is SELECT.

SELECT Statement



SELECT * FROM Student;

The above query displays all records stored in the Student table.

📊 WHERE Clause

The WHERE clause filters records based on a specific condition. 



SELECT *

FROM Student

WHERE Age > 18;

📋 ORDER BY Clause

ORDER BY arranges data in ascending (ASC) or descending (DESC) order. 



SELECT *

FROM Student

ORDER BY Name ASC;

📌 SQL Operators

Operator Description
= Equal To
> Greater Than

⭐ Advantages of SQL

  • ✔ Simple syntax.
  • ✔ Fast data retrieval.
  • ✔ Standard language.
  • ✔ Supports security.
  • ✔ Easy database management.
  • ✔ Supports large databases.

🔐 DCL (Data Control Language)

DCL commands are used to control access to the database. They help the Database Administrator (DBA) provide or remove permissions from users.

📖 Main DCL Commands

  • GRANT – Gives permissions to users.
  • REVOKE – Removes permissions from users.

Example : GRANT



GRANT SELECT, INSERT

ON Student

TO Rahul;

The above command allows the user Rahul to view and insert records into the Student table.

Example : REVOKE



REVOKE INSERT

ON Student

FROM Rahul;

🔄 TCL (Transaction Control Language)

TCL commands are used to manage database transactions. A transaction is a group of SQL operations that are executed together.

Command Purpose
COMMIT Save changes permanently.
ROLLBACK Undo changes.
SAVEPOINT Create a checkpoint.

COMMIT



COMMIT;

Saves all changes permanently in the database.

ROLLBACK



ROLLBACK;

Restores the database to its previous state.

SAVEPOINT



SAVEPOINT SP1;

Creates a checkpoint so that the database can return to that point if needed.

📊 Aggregate Functions

Aggregate Functions perform calculations on multiple rows and return a single result.

Function Description
COUNT() Counts total records.
SUM() Calculates total value.
AVG() Calculates average value.
MIN() Finds minimum value.
MAX() Finds maximum value.

Examples



SELECT COUNT(*) FROM Student;

SELECT AVG(Age) FROM Student;

SELECT MAX(Marks) FROM Student;

SELECT MIN(Marks) FROM Student;

SELECT SUM(Fees) FROM Student;

📂 GROUP BY Clause

GROUP BY is used to group rows having the same values into summary rows. It is often used with aggregate functions. 



SELECT Course,

COUNT(*)

FROM Student

GROUP BY Course;

📌 HAVING Clause

HAVING filters grouped records. It works with GROUP BY. 



SELECT Course,

COUNT(*)

FROM Student

GROUP BY Course

HAVING COUNT(*) > 5;

🔗 SQL Joins

Joins are used to retrieve related data from two or more tables.

Join Description
INNER JOIN Returns matching records.
LEFT JOIN Returns all left table records.
RIGHT JOIN Returns all right table records.
FULL JOIN Returns all matching and non-matching records.

Example : INNER JOIN



SELECT Student.Name,

Course.Course_Name

FROM Student

INNER JOIN Course

ON Student.Course_ID = Course.Course_ID;

📚 Chapter 5 : Normalization

Normalization is the process of organizing data in a database to reduce data redundancy (repetition of data) and improve data consistency. It divides large tables into smaller, related tables and establishes relationships between them using keys. Normalization helps in maintaining data integrity and makes database management more efficient.

📖 Definition

Normalization is a database design technique used to organize data into multiple related tables in order to eliminate redundancy and dependency problems.

🎯 Objectives of Normalization

  • ✔ Remove duplicate data.
  • ✔ Improve data consistency.
  • ✔ Reduce data redundancy.
  • ✔ Simplify database maintenance.
  • ✔ Maintain data integrity.
  • ✔ Improve storage efficiency.

📌 Need of Normalization

Without normalization, the same information may be stored multiple times, leading to redundancy and anomalies. If one value changes, it has to be updated at many places, increasing the chances of inconsistency. Normalization solves these problems by organizing data properly.

⚠ Problems Without Normalization

  • Insertion Anomaly: Difficulty in inserting new data.
  • Update Anomaly: Updating the same data at multiple places.
  • Deletion Anomaly: Deleting one record may remove important information.

🔗 Functional Dependency

A Functional Dependency (FD) exists when the value of one attribute determines the value of another attribute. It is represented using the symbol →. 



Student_ID → Student_Name

Roll_No → Course

Employee_ID → Salary

Here, Student_ID uniquely determines Student_Name.

1️⃣ First Normal Form (1NF)

A table is in First Normal Form if all attribute values are atomic (indivisible) and there are no repeating groups or multiple values in a single column.

Before 1NF

Student Phone Numbers
Rahul 9876543210, 8765432109

After 1NF

Student Phone Number
Rahul 9876543210
Rahul 8765432109

2️⃣ Second Normal Form (2NF)

A table is in Second Normal Form if it is already in 1NF and every non-key attribute is fully dependent on the Primary Key. Partial dependency should not exist.

Example:

Instead of storing Student and Course details in one table, create separate Student and Course tables linked by a key.

3️⃣ Third Normal Form (3NF)

A table is in Third Normal Form if it is already in 2NF and there is no transitive dependency. Non-key attributes should depend only on the Primary Key. 



Employee_ID → Department_ID

Department_ID → Department_Name

Department_Name should be stored in a separate Department table.

4️⃣ Boyce-Codd Normal Form (BCNF)

BCNF is an advanced version of 3NF. It removes remaining anomalies by ensuring that every determinant is a candidate key. BCNF provides a stronger level of normalization.

📊 Comparison of Normal Forms

Normal Form Main Rule
1NF Atomic values only.
2NF No partial dependency.
3NF No transitive dependency.
BCNF Every determinant must be a Candidate Key.

⭐ Advantages of Normalization

  • ✔ Eliminates duplicate data.
  • ✔ Improves database consistency.
  • ✔ Saves storage space.
  • ✔ Simplifies updates and maintenance.
  • ✔ Improves data integrity.

❌ Disadvantages of Normalization

  • ❌ More tables are created.
  • ❌ Complex JOIN operations.
  • ❌ Slight decrease in query performance.

💳 Chapter 6 : Transactions, ACID Properties & Concurrency Control

A transaction is a sequence of one or more database operations that are executed as a single logical unit of work. A transaction may contain INSERT, UPDATE, DELETE, or SELECT statements. The main objective of a transaction is to ensure that the database remains accurate and consistent even if a failure occurs. Transactions are widely used in banking systems, railway reservation, ATM machines, online shopping, and payment gateways.

📖 Definition

A Transaction is a collection of database operations that are performed as one complete unit. A transaction is considered successful only if all operations are completed successfully; otherwise, all changes are cancelled.

🌍 Real-Life Example

Suppose Rahul transfers ₹5,000 from his bank account to Priya's account. This transaction consists of two operations:

  • Debit ₹5,000 from Rahul's account.
  • Credit ₹5,000 to Priya's account.

If the money is debited but not credited due to a system failure, the transaction should be cancelled. This is handled using transactions.

📋 Transaction States

State Description
Active Transaction is currently executing.
Partially Committed Execution completed but not yet saved permanently.
Committed Changes are saved permanently.
Failed Transaction cannot continue due to an error.
Aborted All changes are rolled back.

🔐 ACID Properties

ACID Properties ensure that every database transaction is reliable, accurate, and consistent.

Property Meaning
Atomicity All operations execute completely or none at all.
Consistency Database remains valid before and after the transaction.
Isolation Transactions execute independently.
Durability Committed data remains permanent even after system failure.

1️⃣ Atomicity

Atomicity follows the "All or Nothing" rule. If one operation fails, the entire transaction is cancelled. 



Account A → -5000

Account B → +5000

If credit fails,

Debit is also cancelled.

2️⃣ Consistency

Consistency ensures that the database always follows all rules and constraints. The database remains in a valid state before and after every transaction.

3️⃣ Isolation

Isolation ensures that multiple transactions running at the same time do not interfere with each other.

4️⃣ Durability

After a transaction is committed, the changes become permanent and cannot be lost even if the system crashes.

⚡ Concurrency Control

Concurrency Control is the process of managing multiple transactions executing simultaneously without causing data inconsistency. It ensures that all users can access the database safely at the same time.

🚨 Problems in Concurrent Transactions

Dirty Read

A transaction reads data that has not yet been committed by another transaction. If the first transaction is rolled back, the second transaction has read invalid data.

Lost Update

Two users update the same record simultaneously, causing one update to overwrite the other.

Non-Repeatable Read

A transaction reads the same row twice but gets different values because another transaction modified the data in between.

Phantom Read

A transaction executes the same query twice but gets different numbers of rows because another transaction inserted or deleted records.

🔒 Locking Techniques

Locking prevents multiple users from modifying the same data at the same time. It helps maintain data consistency.

Lock Type Description
Shared Lock Allows only reading of data.
Exclusive Lock Allows reading and writing; others cannot modify the data.

💾 Recovery Techniques

Recovery is the process of restoring the database after a failure. It ensures that committed transactions are preserved and incomplete transactions are rolled back.

Recovery Methods

  • ✔ Backup and Restore
  • ✔ Log Files
  • ✔ Checkpoints
  • ✔ Rollback
  • ✔ Roll Forward
